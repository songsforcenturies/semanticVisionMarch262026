**Key Technical Concepts**
-   **User Impersonation:** An admin endpoint generates a short-lived JWT for another user. The frontend stores the original admin token, uses the temporary user token for API calls, and provides an exit mechanism to restore the admin session.
-   **Centralized API Key Management:** Storing third-party secrets (Stripe, PayPal, Daily.co) in a database collection, managed via an admin UI. The application logic is designed to prioritize these database keys with a fallback to  variables.
-   **Third-Party Integration (Daily.co):** Integrating a real-time communication API for screen sharing. Involves creating backend endpoints to generate unique room URLs and a frontend interface for admins to initiate support sessions.
