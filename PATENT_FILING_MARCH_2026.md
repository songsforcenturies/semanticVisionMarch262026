## BACKGROUND OF THE INVENTION

### Technical Problem

The field of educational technology ("EdTech") faces five persistent and interrelated challenges:

**1. Static, One-Size-Fits-All Content.** Existing vocabulary and reading platforms (e.g., Duolingo, Khan Academy, Reading IQ, ABCmouse, Readability Tutor, Amira Learning) rely on pre-authored, static content libraries. These platforms cannot dynamically personalize educational narratives to reflect a student's individual religion, cultural heritage, specific interests, character development goals, personal strengths, or areas of growth. The content is authored once and served identically to all students, regardless of their backgrounds. While some adaptive reading platforms (Readability Tutor, EPS Reading Assistant) adjust difficulty levels, none generate entirely new narrative content personalized across multiple dimensions simultaneously.

**2. Intrusive, Educationally Disruptive Monetization.** The predominant monetization models in EdTech — banner advertisements, interstitial video ads, and hard paywalls — interrupt the learning experience. Banner ads are cognitively distracting. Video interstitials break the narrative flow. Paywalls create equity gaps. No existing system integrates brand messaging *within* educational content in a way that adds educational value rather than detracting from it.

**3. Absence of Measurable Brand-Education Feedback Loops.** Even in platforms that accept advertising, brands receive only surface-level metrics (click-through rates, impressions counted by page loads). No existing system provides brands with granular analytics showing: (a) the exact narrative context in which their product appeared, (b) specific comprehension questions generated about their product integration, (c) student pass/fail rates on those questions, or (d) the full text of the educational content where the product was featured.

**4. Fragmented Stakeholder Ecosystems.** Existing educational platforms typically serve one or two user types. No single platform provides purpose-built portals for five distinct stakeholder roles (administrators, parents/guardians, teachers, students, and brand partners) with granular permission controls, each with their own dashboard, analytics, and feature sets.

**5. Absence of Cultural and Faith-Based Content Personalization.** While AI-powered educational tools exist for Christian schools and homeschooling (as described in various publications), no patented system generates educational narratives that are simultaneously personalized for specific belief systems (Christianity, Islam, Buddhism, Hinduism, Judaism, Secular Humanism, etc.) and cultural contexts (African American, South Asian, Latin American, etc.) using LLM-driven content generation.

## SUMMARY OF THE INVENTION

The present invention, commercially embodied as "Semantic Vision," provides a computer-implemented platform comprising:

**A. Content Generation Pipeline:** A real-time pipeline that (1) assembles a multi-dimensional student profile, (2) selects vocabulary according to a novel 60/30/10 three-tier distribution model, (3) executes a real-time brand eligibility engine, (4) constructs a composite AI prompt incorporating all personalization dimensions, (5) directs a Large Language Model to generate structured multi-chapter educational narratives, (6) records brand impressions and updates budgets in real time, and (7) generates context-aware comprehension questions.

**B. Assessment and Mastery Pipeline:** A secondary AI pipeline that (1) evaluates student vocabulary responses using LLM-based assessment, (2) tracks vocabulary mastery with normalized token sets, and (3) computes an Agentic Reach Score measuring holistic learning progress.

**C. Closed-Loop Brand Analytics:** A multi-layer search and analytics engine that (1) identifies all narratives containing brand content through three-layer discovery, (2) extracts specific story excerpts, (3) identifies brand-related comprehension questions, (4) retrieves student interaction data, and (5) presents comprehensive engagement analytics to brand partners.

**D. Multi-Role Platform Architecture:** A five-tier role-based access control system providing purpose-built portals for Administrators, Parents/Guardians, Teachers, Students, and Brand Partners.

**E. Affiliate Referral Engine:** A closed-loop referral system with auto-generated affiliate codes, configurable reward structures (flat fee, percentage, wallet credits), automated email confirmation, and referral tracking integrated into the registration flow.

**F. Guided Onboarding Framework:** Role-specific, skippable onboarding wizards triggered on first login, with client-side state persistence and re-triggerable tutorial functionality.

**G. Unified Multi-Role Authentication Interface:** A single authentication interface with a role selector that dynamically switches between authentication modalities (email/password for adults, code/PIN for students) based on the selected role.

## DETAILED DESCRIPTION OF THE PREFERRED EMBODIMENTS

### 1. System Architecture Overview (FIG. 13)

The preferred embodiment comprises a distributed computing system with the following components:

**1.1 Backend Application Server.** A Python-based asynchronous web application server built on the FastAPI framework, providing RESTful API endpoints for all system operations. The server implements role-based access control ("RBAC") using JSON Web Tokens ("JWT") for five distinct user roles: Administrator, Guardian/Parent, Teacher, Student, and Brand Partner.

**1.2 Database Layer.** A MongoDB document-oriented database accessed via the Motor asynchronous driver, storing all user profiles, student records, word banks, brand records, narratives, assessments, brand impressions, read logs, written answers, affiliate records, brand offers, and system configuration.

**1.3 AI Engine.** A Large Language Model integration layer supporting multiple providers (including OpenAI GPT-5.2 and configurable alternatives), used for two distinct functions: (a) multi-chapter educational narrative generation, and (b) student response evaluation for both comprehension questions and vocabulary assessments.

**1.4 Frontend Client Application.** A React 18 single-page application with role-specific portals for each user type, providing the user interface for all system interactions including guided onboarding, story reading, progress tracking, brand campaign management, and administrative controls.

**1.5 Payment Processing Layer.** A Stripe API integration providing wallet top-up functionality for guardians and budget management for brand partners.

**1.6 Email Service Layer.** A Resend API integration for transactional emails including password reset, affiliate confirmation, and referral notifications.

**1.7 Geolocation and Currency Layer.** IP-based geolocation (ip-api.com) and real-time currency conversion (Open Exchange Rates API) supporting 50+ currencies for global accessibility.

**1.8 Internationalization Layer.** A react-i18next integration supporting 20+ languages for the entire user interface, with the AI engine generating narrative content in the student's preferred language.

### 2. Multi-Dimensional Student Profile Assembly (FIG. 14, Step 1)

The system maintains a comprehensive student profile record comprising the following data fields, each of which contributes to the personalization of generated content:

**2.1 Demographic Data.** Full name, age (integer), and grade level (enumerated: pre-kindergarten, kindergarten, grades 1-12, college, adult).

**2.2 Interest Graph.** A list of student interests (e.g., "soccer," "robots," "animals") used to theme generated narratives.

**2.3 Character Education Virtues.** A selectable list of 16 virtues (Courage, Honesty, Kindness, Perseverance, Respect, Responsibility, Compassion, Generosity, Gratitude, Humility, Integrity, Justice, Patience, Self-Discipline, Tolerance, Wisdom) with descriptive text for each, that the AI is instructed to model through character behavior in the narrative.

**2.4 Belief System Identifier.** A string identifier (e.g., "Christianity," "Islam," "Buddhism," "Secular Humanism") that directs the AI to reflect corresponding values, teachings, and moral frameworks.

**2.5 Cultural Context Identifier.** A string identifier (e.g., "African American," "South Asian," "Latin American") that directs the AI to incorporate culturally relevant names, settings, traditions, foods, and customs.

**2.6 Language Preference.** A language identifier supporting 20+ languages, directing the AI to generate the entire narrative in the specified language.

**2.7 Strengths Description.** A free-text field authored by the student's parent describing the child's strengths. The AI celebrates these as "superpowers" through protagonist behavior.

**2.8 Weaknesses/Growth Areas Description.** A free-text field authored by the parent. The AI models growth through empathetic character development, never through shame or deficit framing.

**2.9 Advertising Preferences.** `allow_brand_stories` (Boolean, default false requiring affirmative opt-in) and `blocked_categories` (List of Strings for granular category blocking).

**2.10 Vocabulary State.** Assigned word bank identifiers, mastered vocabulary tokens (normalized lowercase strings), and Agentic Reach Score (float, 0-100).

### 3. Three-Tier Vocabulary Distribution System (60/30/10 Model)

The present invention implements a novel vocabulary distribution model:

**3.1** Retrieves all assigned word banks, each containing `baseline_words`, `target_words`, and `stretch_words` with definitions and example sentences.

**3.2** Aggregates into three master lists and randomly shuffles each for non-repeating sequences.

**3.3** Selects according to the 60/30/10 distribution:
- **60% Baseline Words (18 max):** Foundation-level vocabulary for mastery reinforcement.
- **30% Target Words (9 max):** Growth-level vocabulary as the primary learning objective.
- **10% Stretch Words (3 max):** Challenge-level vocabulary for aspirational engagement.

### 4. Age-Calibrated Vocabulary Complexity System

The present invention implements a novel 16-level vocabulary complexity calibration system, specifying exact language parameters for each grade level:

| Grade Level | Sentence Length | Vocabulary Complexity |
|---|---|---|
| Pre-K (Ages 3-4) | 3-5 words | Sight words, phonics-friendly, simple CVC words |
| Kindergarten (Ages 5-6) | 5-8 words | Common sight words, simple sentences |
| Grade 1 (Ages 6-7) | 6-10 words | Dolch sight words, simple compound sentences |
| Grade 2 (Ages 7-8) | 8-12 words | Grade-level vocabulary, basic figurative language |
| Grades 3-4 (Ages 8-10) | 10-15 words | Multi-syllable words, context clues |
| Grades 5-6 (Ages 10-12) | 12-18 words | Academic vocabulary, literary devices |
| Grades 7-8 (Ages 12-14) | 15-20 words | Abstract concepts, domain-specific terminology |
| Grades 9-10 (Ages 14-16) | 15-25 words | SAT-prep vocabulary, complex syntax |
| Grades 11-12 (Ages 16-18) | 18-30 words | AP-level vocabulary, nuanced argumentation |
| College (Ages 18+) | 20-35 words | Discipline-specific jargon, critical analysis |
| Adult | Unlimited | Professional and technical vocabulary |

This calibration ensures that all generated content — including brand integration prose, comprehension questions, and vocabulary context sentences — is accessible to the student at their developmental level.

### 5. Real-Time Brand Eligibility Engine (FIG. 15)

At each content generation request, the system executes a multi-factor filtering pipeline:

**5.1 System-Level Feature Gate.** Checks a globally configurable feature flag (`brand_sponsorship_enabled`).

**5.2 Guardian Consent Check.** Reads `ad_preferences.allow_brand_stories` (default `false`, requiring affirmative opt-in).

**5.3 Active Brand Query.** Queries the brand database for all active brands.

**5.4 Age Appropriateness Filter.** Checks each brand's `target_ages` against the student's age. Brands without age restrictions default to "every market."

**5.5 Budget Availability Check.** Verifies brand's `budget_spent` has not reached `budget_total`.

**5.6 Category Block List Check.** Cross-references brand `target_categories` against guardian's `blocked_categories`.

**5.7 Selection and Limit.** Eligible brands are selected up to a configurable maximum (default: 2 per narrative).

**5.8 Dynamic Composition.** Every narrative has a potentially unique brand composition based on live database state.

### 6. Composite AI Prompt Construction

The system synthesizes all personalization dimensions into a single LLM prompt comprising:

- Student profile data (name, age, grade, interests, virtues)
- Tiered vocabulary distribution (60/30/10)
- Belief system alignment directive
- Cultural context integration directive
- Language directive (20+ languages)
- Brand integration directive (products as organic problem-solving elements)
- Strengths celebration directive
- Growth areas empathy directive
- Age-calibrated vocabulary complexity specifications
- Structural requirements (5 chapters, 300-500 words each)
- Output JSON schema

### 7. AI Story Generation and Post-Processing

**7.1 LLM Invocation.** Composite prompt transmitted to configurable LLM (default: OpenAI GPT-5.2).

**7.2 Response Parsing and Validation.** JSON parsing, structural validation, vocabulary token verification, comprehension question validation.

**7.3 Narrative Persistence.** Storage with all chapter content, tokens, questions, and brand placement data.

**7.4 Brand Impression Recording.** Creates impression records, assigns cost-per-impression, atomically updates brand counters.

**7.5 Cost Logging.** Records model, tokens, cost, duration, and success status per generation event.

### 8. Student Interaction and Assessment Pipeline

**8.1 Narrative Reading.** Chapter-by-chapter reading with per-chapter comprehension questions.

**8.2 Vocabulary Assessment Generation.** Questions for target-tier and stretch-tier words.

**8.3 AI-Powered Assessment Evaluation.** Second LLM invocation evaluating definition accuracy, contextual usage, and spelling (exact or phonetic modes).

**8.4 Written Answer Evaluation.** Free-text comprehension responses evaluated for understanding, relevance, and spelling.

**8.5 Mastery Tracking.** At 80%+ accuracy, mastered tokens are normalized (lowercase), merged with existing set, and Agentic Reach Score is recalculated:

```
Agentic Reach Score = min(
    (|mastered_tokens| × 10 + completed_narratives × 50) /
    max(total_narratives × 50, 1) × 100,
    100.0
)
```

### 9. Closed-Loop Brand Analytics Pipeline (FIG. 16)

**9.1 Three-Layer Narrative Discovery.**
- Layer 1: Structured brand placement queries
- Layer 2: Impression record cross-referencing
- Layer 3: Full-text content search

**9.2 Story Excerpt Extraction.** Sentence-level extraction of brand mentions with up to 5 excerpts per chapter.

**9.3 Brand Activation Question Identification.** Four-condition classification:
- Condition A: Direct brand/product mention in question/answer
- Condition B: Product category keyword + brand chapter context
- Condition C: Answer contains brand product name derivatives
- Condition D: Contextual keyword proximity to brand mentions

**9.4 Question Performance Analytics.** Per-question tallies of attempts, passes, and failures.

**9.5 Summary Statistics.** Total stories, mentions, questions, attempts, pass rate, unique students, average comprehension score.

**9.6 Full Story Reader.** Brand partners can read complete narratives with brand-mention indicators.

### 10. Consent Architecture and Privacy Controls

**10.1 Parental Consent Gate.** Default-false opt-in. Revocable at any time.

**10.2 Granular Category Blocking.** Per-category brand exclusion.

**10.3 Age-Appropriate Filtering.** Automatic brand exclusion based on target ages.

**10.4 Budget-Limited Exposure.** Automatic exclusion when brand budget exhausted.

**10.5 Student Authentication Isolation.** Code/PIN authentication (not email/password), minimizing PII collection.

### 12. Affiliate Referral System

The present invention includes a novel affiliate referral engine comprising:

**12.1 Public Affiliate Signup.** Any registered user can request affiliate status. The system auto-generates a unique affiliate code (format: AFF-XXXXXX) and sends an automated confirmation email via the Resend API containing the affiliate link.

**12.2 Configurable Reward Structures.** Administrators can set the reward type for the affiliate program:
- **Flat Fee:** A fixed dollar amount per successful referral sign-up.
- **Percentage:** A percentage of the referred user's first purchase.
- **Wallet Credits:** A fixed amount deposited directly into the affiliate's platform wallet.

**12.3 Referral Tracking.** When a new user registers with an affiliate code (AFF-XXXXXX), the system:
- Validates the affiliate code against the affiliate database
- Records the new user's ID in the affiliate's `signups` array
- Increments the affiliate's referral count
- Calculates and records the earned reward
- Updates the affiliate's pending payout balance

**12.4 Administrative Controls.** Administrators can:
- Set default reward type and values (flat fee amount, percentage rate, wallet credit amount)
- Set minimum payout threshold
- Toggle program active/inactive
- Toggle auto-approve for new affiliates
- View all affiliates with their codes, referral counts, earned amounts, and pending payouts
- Approve, deactivate, or record payouts for individual affiliates

**Claim 1.** A computer-implemented method for generating personalized educational content with integrated brand product placements, comprising:

(a) receiving, by a processor, a content generation request associated with a student identifier;

(b) retrieving, from a database, a student profile comprising at least: age, grade level, interests, a belief system identifier, a cultural context identifier, a language preference, advertising preferences including a brand content opt-in flag and a blocked categories list, and at least one assigned word bank identifier;

(c) retrieving vocabulary data from each assigned word bank, comprising baseline words, target words, and stretch words;

(d) selecting vocabulary according to a tiered distribution comprising approximately 60% baseline words, approximately 30% target words, and approximately 10% stretch words, with randomized selection;

(e) determining, based on the advertising preferences, whether brand integration is authorized;

(f) when authorized, executing a real-time brand eligibility engine comprising:
   (i) querying the database for active brand records;
   (ii) filtering based on: student age relative to brand target age range, student blocked categories relative to brand target categories, and brand remaining campaign budget;
   (iii) selecting eligible brands not exceeding a configurable maximum;

(g) constructing a composite prompt for a Large Language Model incorporating: the student profile, selected tiered vocabulary, belief system identifier, cultural context identifier, language preference, age-calibrated vocabulary complexity specifications, character education directives, and brand integration directives specifying that products are to be integrated as organic, problem-solving narrative elements;

(h) transmitting the composite prompt to the LLM and receiving a structured response comprising a multi-chapter educational narrative with embedded vocabulary annotations and per-chapter comprehension questions;

(i) storing the generated narrative with associated brand placement data; and

(j) recording brand impression records and updating brand campaign budgets.

(d) identifying brand-related comprehension questions based on: direct brand/product name appearance, product category context, answer proximity, or contextual relevance;

## ABSTRACT

A computer-implemented system and method for generating personalized, AI-driven educational narratives that dynamically integrate brand product placements as organic, problem-solving story elements. The system assembles a multi-dimensional student profile comprising demographic, cultural, religious, linguistic, developmental, and interest-based data. At each content generation request, a real-time brand eligibility engine queries a live brand marketplace database and applies multi-factor filtering based on student age, guardian consent, category restrictions, and brand budget availability. A composite prompt synthesizes the student profile, a three-tier (60/30/10) vocabulary distribution, age-calibrated vocabulary complexity across 16 grade levels, and brand integration directives for a Large Language Model, which generates a structured multi-chapter educational narrative. The system records brand impressions and updates campaign budgets in real time. A closed-loop analytics pipeline identifies narratives containing brand content through multi-layer search, extracts story excerpts with brand mentions, identifies brand-related comprehension questions, and presents comprehensive engagement analytics to brand partners. The platform implements five-tier role-based access control with purpose-built portals for administrators, parents, teachers, students, and brand partners. Additional innovations include: an affiliate referral engine with configurable reward structures (flat fee, percentage, wallet credits), guided role-specific onboarding wizards with tutorial reset capability, a unified multi-role authentication interface with dynamic form switching, a contextual FAQ system, a multi-step student creation wizard, real-time classroom sessions via WebSocket, and multi-currency support across 50+ currencies. The consent architecture requires affirmative guardian opt-in for brand content with granular category blocking. An Agentic Reach Score provides holistic measurement of vocabulary acquisition relative to engagement volume.
